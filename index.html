01
Two Sum
HashMap CitiHSBCBarclaysGS
Problem: Given an integer array nums and an integer target, return indices of the two numbers such that they add up to target. Each input has exactly one solution. You may not use the same element twice.
Brute Force
Check every pair of elements using two nested loops. For each element, iterate through all subsequent elements to find the complement.
TC: O(n²)SC: O(1)
java
public int[] twoSum(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] == target)
                return new int[]{i, j};
        }
    }
    return new int[]{};
}
Optimized (HashMap)
Use a HashMap to store each element's value→index as we traverse. For each element, check if its complement (target - nums[i]) already exists in the map.
TC: O(n)SC: O(n)
java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement))
            return new int[]{map.get(complement), i};
        map.put(nums[i], i);
    }
    return new int[]{};
}
02
Longest Substring Without Repeating Characters
Sliding Window BankingProduct
Problem: Given a string s, find the length of the longest substring without repeating characters.
Brute Force
Generate all substrings using two loops. For each substring, use a HashSet to check if all characters are unique. Track the maximum length found.
TC: O(n³)SC: O(n)
java
public int lengthOfLongestSubstring(String s) {
    int max = 0;
    for (int i = 0; i < s.length(); i++) {
        Set<Character> set = new HashSet<>();
        for (int j = i; j < s.length(); j++) {
            if (set.contains(s.charAt(j))) break;
            set.add(s.charAt(j));
            max = Math.max(max, j - i + 1);
        }
    }
    return max;
}
Optimized (Sliding Window + HashMap)
Use two pointers (left, right) and a HashMap storing char→last index. Expand right pointer; when duplicate found, move left pointer to max(left, lastIndex+1). Update max length.
TC: O(n)SC: O(min(m,n))
java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int max = 0, left = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c))
            left = Math.max(left, map.get(c) + 1);
        map.put(c, right);
        max = Math.max(max, right - left + 1);
    }
    return max;
}
03
LRU Cache
HashMap + Doubly LinkedList GSFinance
Problem: Design a data structure that follows the Least Recently Used cache constraint. Implement get(key) and put(key, value) in O(1). Evict the least recently used item when capacity is reached.
Brute Force (LinkedHashMap shortcut)
Use Java's LinkedHashMap with access-order=true and override removeEldestEntry. Quick implementation but doesn't show deep understanding expected in interviews.
TC: O(1) avgSC: O(capacity)
java
class LRUCache extends LinkedHashMap<Integer, Integer> {
    private final int cap;
    public LRUCache(int capacity) {
        super(capacity, 0.75f, true);
        this.cap = capacity;
    }
    public int get(int key) {
        return getOrDefault(key, -1);
    }
    public void put(int key, int value) {
        super.put(key, value);
    }
    protected boolean removeEldestEntry(Map.Entry e) {
        return size() > cap;
    }
}
Optimized (HashMap + Doubly Linked List)
HashMap gives O(1) key lookup. Doubly linked list maintains access order — most recent at head, LRU at tail. On get/put: move node to head. On evict: remove tail. Follow-up: use ReentrantReadWriteLock for thread safety.
TC: O(1)SC: O(capacity)
java
class LRUCache {
    private class Node {
        int key, val;
        Node prev, next;
        Node(int k, int v) { key = k; val = v; }
    }
    private Map<Integer, Node> map = new HashMap<>();
    private Node head, tail;
    private int cap;

    public LRUCache(int capacity) {
        cap = capacity;
        head = new Node(0, 0); // dummy head
        tail = new Node(0, 0); // dummy tail
        head.next = tail; tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        remove(node);
        insertToHead(node);
        return node.val;
    }

    public void put(int key, int val) {
        if (map.containsKey(key)) remove(map.get(key));
        Node node = new Node(key, val);
        map.put(key, node);
        insertToHead(node);
        if (map.size() > cap) {
            Node lru = tail.prev;
            remove(lru);
            map.remove(lru.key);
        }
    }

    private void remove(Node n) {
        n.prev.next = n.next; n.next.prev = n.prev;
    }
    private void insertToHead(Node n) {
        n.next = head.next; n.prev = head;
        head.next.prev = n; head.next = n;
    }
}

Merge Intervals
Sorting + Greedy SchedulingTrade Window
Problem: Given an array of intervals where intervals[i] = [start, end], merge all overlapping intervals and return an array of the non-overlapping intervals.
Brute Force
For each interval, compare with all others to find overlaps and merge repeatedly until no more merges are possible. Inefficient due to repeated passes.
TC: O(n²)SC: O(n)
java
// Simplified brute: sort + single pass (still considered brute without greedy insight)
public int[][] merge(int[][] intervals) {
    List<int[]> res = new ArrayList<>();
    for (int[] a : intervals) {
        boolean merged = false;
        for (int[] r : res) {
            if (a[0] <= r[1] && a[1] >= r[0]) {
                r[0] = Math.min(r[0], a[0]);
                r[1] = Math.max(r[1], a[1]);
                merged = true; break;
            }
        }
        if (!merged) res.add(new int[]{a[0], a[1]});
    }
    return res.toArray(new int[0][]);
}
Optimized (Sort + Greedy)
Sort intervals by start time. Iterate: if current interval overlaps with last in result (curr.start ≤ last.end), merge by extending end. Otherwise, add as new interval.
TC: O(n log n)SC: O(n)
java
public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    List<int[]> res = new ArrayList<>();
    for (int[] curr : intervals) {
        if (res.isEmpty() || res.get(res.size()-1)[1] < curr[0]) {
            res.add(curr);
        } else {
            res.get(res.size()-1)[1] = Math.max(res.get(res.size()-1)[1], curr[1]);
        }
    }
    return res.toArray(new int[0][]);
}
05
Meeting Rooms II (Max Concurrent Meetings)
Interval SchedulingHeap GSBackend
Problem: Given an array of meeting time intervals [start, end], find the minimum number of conference rooms required.
Brute Force
For each meeting, check how many other meetings overlap with it. The maximum overlap count is the answer. Compare every pair.
TC: O(n²)SC: O(1)
java
public int minMeetingRooms(int[][] intervals) {
    int max = 0;
    for (int i = 0; i < intervals.length; i++) {
        int count = 1;
        for (int j = 0; j < intervals.length; j++) {
            if (i != j && intervals[j][0] < intervals[i][1]
                       && intervals[j][1] > intervals[i][0])
                count++;
        }
        max = Math.max(max, count);
    }
    return max;
}
Optimized (Min-Heap)
Sort by start time. Use a min-heap storing end times of ongoing meetings. For each new meeting, if heap's min end ≤ current start, reuse that room (poll). Always push current end. Heap size = rooms needed.
TC: O(n log n)SC: O(n)
java
public int minMeetingRooms(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
    PriorityQueue<Integer> heap = new PriorityQueue<>(); // min-heap of end times
    for (int[] interval : intervals) {
        if (!heap.isEmpty() && heap.peek() <= interval[0])
            heap.poll(); // reuse room
        heap.offer(interval[1]);
    }
    return heap.size();
}
06
Reverse Linked List
Pointer Manipulation Mid-Level Classic
Problem: Given the head of a singly linked list, reverse the list and return the reversed list.
Iterative (Standard)
Use three pointers: prev=null, curr=head, next. In each iteration, save next, reverse curr.next to prev, advance prev and curr forward.
TC: O(n)SC: O(1)
java
public ListNode reverseList(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
Recursive
Recursively reach the last node (new head), then on the way back fix pointers: head.next.next = head; head.next = null. Elegant but uses O(n) stack space.
TC: O(n)SC: O(n) stack
java
public ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseList(head.next);
    head.next.next = head;
    head.next = null;
    return newHead;
}

Detect Cycle in Linked List
Floyd's Cycle Detection Follow-up: Find Start
Problem: Given head of linked list, determine if it has a cycle. Follow-up: Return the node where the cycle begins.
Brute Force (HashSet)
Traverse the list, storing each visited node in a HashSet. If we encounter a node already in the set, a cycle exists. Return that node as cycle start.
TC: O(n)SC: O(n)
java
public boolean hasCycle(ListNode head) {
    Set<ListNode> visited = new HashSet<>();
    ListNode curr = head;
    while (curr != null) {
        if (!visited.add(curr)) return true;
        curr = curr.next;
    }
    return false;
}
Optimized (Floyd's Two Pointers)
Slow moves 1 step, fast moves 2 steps. If they meet, cycle exists. To find start: reset slow to head, keep fast at meeting point, advance both 1 step — they meet at cycle start (mathematical proof).
TC: O(n)SC: O(1)
java
// Detect cycle
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}

// Find cycle start node
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next; fast = fast.next.next;
        if (slow == fast) {
            slow = head;
            while (slow != fast) { slow = slow.next; fast = fast.next; }
            return slow;
        }
    }
    return null;
}

Kth Largest Element in an Array
HeapQuickSelect LeaderboardTop Traders
Problem: Given an integer array nums and integer k, return the kth largest element in the array (not kth distinct).
Brute Force (Sort)
Sort the array in descending order and return the element at index k-1. Simple but not optimal for large datasets.
TC: O(n log n)SC: O(1)
java
public int findKthLargest(int[] nums, int k) {
    Arrays.sort(nums);
    return nums[nums.length - k];
}
Better (Min-Heap of size k)
Maintain a min-heap of size k. For each element: if heap size < k, add it. Else if element > heap.peek(), replace. Final heap.peek() is the kth largest.
TC: O(n log k)SC: O(k)
java
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int num : nums) {
        minHeap.offer(num);
        if (minHeap.size() > k) minHeap.poll();
    }
    return minHeap.peek();
}
Optimized (QuickSelect)
Partition array like QuickSort around a pivot. If pivot index == n-k, return it. If pivot is too right, recurse left; if too left, recurse right. Average O(n) due to halving search space.
TC: O(n) avg, O(n²) worstSC: O(1)
java
public int findKthLargest(int[] nums, int k) {
    return quickSelect(nums, 0, nums.length - 1, nums.length - k);
}
private int quickSelect(int[] a, int lo, int hi, int target) {
    int pivot = a[hi], p = lo;
    for (int i = lo; i < hi; i++)
        if (a[i] <= pivot) { int t = a[p]; a[p] = a[i]; a[i] = t; p++; }
    int t = a[p]; a[p] = a[hi]; a[hi] = t;
    if (p == target) return a[p];
    return p < target ? quickSelect(a, p+1, hi, target) : quickSelect(a, lo, p-1, target);
}

Top K Frequent Elements
Heap + HashMap Log AnalysisMarket Symbols
Problem: Given an integer array nums and integer k, return the k most frequent elements.
Brute Force (Sort by frequency)
Build frequency map, then sort entries by value descending. Take first k keys. Simple but slower than heap-based approach.
TC: O(n log n)SC: O(n)
java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer,Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);
    return freq.entrySet().stream()
        .sorted((a,b) -> b.getValue() - a.getValue())
        .limit(k).mapToInt(Map.Entry::getKey).toArray();
}
Better (Min-Heap of size k)
Build frequency map. Use min-heap of size k keyed by frequency. Maintains only the top-k, saving memory and time over full sort.
TC: O(n log k)SC: O(n)
java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer,Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);
    PriorityQueue<Integer> heap = new PriorityQueue<>((a,b) -> freq.get(a)-freq.get(b));
    for (int key : freq.keySet()) {
        heap.offer(key);
        if (heap.size() > k) heap.poll();
    }
    return heap.stream().mapToInt(Integer::intValue).toArray();
}
Optimized (Bucket Sort)
Build frequency map. Create buckets array of size n+1 where index = frequency. Iterate buckets from end to collect top-k elements. O(n) total.
TC: O(n)SC: O(n)
java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer,Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);
    List<Integer>[] bucket = new List[nums.length + 1];
    for (int key : freq.keySet()) {
        int f = freq.get(key);
        if (bucket[f] == null) bucket[f] = new ArrayList<>();
        bucket[f].add(key);
    }
    List<Integer> res = new ArrayList<>();
    for (int i = bucket.length-1; i >= 0 && res.size() < k; i--)
        if (bucket[i] != null) res.addAll(bucket[i]);
    return res.stream().mapToInt(Integer::intValue).toArray();
}

Binary Tree Level Order Traversal
BFS Queue Finance Firms
Problem: Given root of a binary tree, return the level order traversal of its nodes' values (i.e., left to right, level by level).
Recursive DFS
Use DFS with a depth parameter. Maintain a result list of lists. At each node, add value to result[depth]. Recurse left then right.
TC: O(n)SC: O(n)
java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    dfs(root, 0, res);
    return res;
}
private void dfs(TreeNode node, int depth, List<List<Integer>> res) {
    if (node == null) return;
    if (res.size() == depth) res.add(new ArrayList<>());
    res.get(depth).add(node.val);
    dfs(node.left, depth+1, res);
    dfs(node.right, depth+1, res);
}
Optimized (BFS Queue)
Use a Queue. Add root, then process level-by-level: capture queue size at start of each level, process exactly that many nodes, add children to queue. Cleanest BFS pattern.
TC: O(n)SC: O(n)
java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
    }
    return res;
}

Lowest Common Ancestor (Binary Tree)
Tree Recursion GSSprinklr
Problem: Given a binary tree and two nodes p and q, find their Lowest Common Ancestor — the deepest node that has both p and q as descendants.
Brute Force (Path Tracking)
Find root-to-p path and root-to-q path separately. Compare both paths and return the last common node before they diverge.
TC: O(n)SC: O(n)
java
private boolean findPath(TreeNode root, TreeNode target, List<TreeNode> path) {
    if (root == null) return false;
    path.add(root);
    if (root == target) return true;
    if (findPath(root.left, target, path) || findPath(root.right, target, path))
        return true;
    path.remove(path.size()-1);
    return false;
}
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    List<TreeNode> pathP = new ArrayList<>(), pathQ = new ArrayList<>();
    findPath(root, p, pathP); findPath(root, q, pathQ);
    TreeNode lca = null;
    for (int i = 0; i < Math.min(pathP.size(), pathQ.size()); i++) {
        if (pathP.get(i) == pathQ.get(i)) lca = pathP.get(i);
        else break;
    }
    return lca;
}
Optimized (Single DFS)
If current node is p or q, return it. Recurse left and right. If both return non-null, current node is LCA. If only one returns non-null, propagate that up.
TC: O(n)SC: O(h)
java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left  = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;
    return left != null ? left : right;
}

Validate Binary Search Tree
DFS + Range Validation GS
Problem: Given root of binary tree, determine if it is a valid Binary Search Tree (BST). A valid BST: left subtree has only nodes less than root, right subtree greater, both subtrees also valid BSTs.
Brute Force (Inorder traversal check)
Inorder traversal of a BST gives sorted sequence. Collect all values via inorder, then check if the list is strictly increasing.
TC: O(n)SC: O(n)
java
public boolean isValidBST(TreeNode root) {
    List<Integer> vals = new ArrayList<>();
    inorder(root, vals);
    for (int i = 1; i < vals.size(); i++)
        if (vals.get(i) <= vals.get(i-1)) return false;
    return true;
}
private void inorder(TreeNode n, List<Integer> vals) {
    if (n == null) return;
    inorder(n.left, vals); vals.add(n.val); inorder(n.right, vals);
}
Optimized (DFS with bounds)
Pass min and max bounds down recursively. Each node must be strictly between its bounds. Left child gets max=current.val; right child gets min=current.val. Early termination on violation.
TC: O(n)SC: O(h)
java
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}
private boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    return validate(node.left, min, node.val) &&
           validate(node.right, node.val, max);
}

Number of Islands
Graph DFS/BFS Fraud ClustersNetwork Connectivity
Problem: Given a 2D grid of '1's (land) and '0's (water), count the number of islands. An island is surrounded by water and formed by connecting adjacent land horizontally/vertically.
Optimized (DFS)
Iterate grid. When '1' found, increment count and DFS to mark all connected land cells as '0' (visited). DFS explores all 4 directions. Each DFS call sinks an entire island.
TC: O(m×n)SC: O(m×n)
java
public int numIslands(char[][] grid) {
    int count = 0;
    for (int i = 0; i < grid.length; i++)
        for (int j = 0; j < grid[0].length; j++)
            if (grid[i][j] == '1') { dfs(grid, i, j); count++; }
    return count;
}
private void dfs(char[][] g, int r, int c) {
    if (r < 0 || r >= g.length || c < 0 || c >= g[0].length || g[r][c] != '1') return;
    g[r][c] = '0';
    dfs(g,r+1,c); dfs(g,r-1,c); dfs(g,r,c+1); dfs(g,r,c-1);
}
Alternative (BFS)
Same logic with BFS using a Queue. For each unvisited '1', BFS explores and marks all connected cells. Avoids recursion stack issues for very large grids.
TC: O(m×n)SC: O(min(m,n))
java
public int numIslands(char[][] grid) {
    int count = 0, rows = grid.length, cols = grid[0].length;
    int[] dr = {0,0,1,-1}, dc = {1,-1,0,0};
    for (int i = 0; i < rows; i++) for (int j = 0; j < cols; j++) {
        if (grid[i][j] != '1') continue;
        count++;
        Queue<int[]> q = new LinkedList<>();
        q.offer(new int[]{i,j}); grid[i][j] = '0';
        while (!q.isEmpty()) {
            int[] cur = q.poll();
            for (int d = 0; d < 4; d++) {
                int nr = cur[0]+dr[d], nc = cur[1]+dc[d];
                if (nr>=0&&nr<rows&&nc>=0&&nc<cols&&grid[nr][nc]=='1')
                    { q.offer(new int[]{nr,nc}); grid[nr][nc]='0'; }
            }
        }
    }
    return count;
}

Course Schedule (Cycle in Directed Graph)
Topological SortDFS Dependency Problems
Problem: There are numCourses labeled 0 to n-1. Given prerequisites[i] = [a, b] (must take b before a), return true if you can finish all courses (no cycle in directed graph).
DFS Cycle Detection (3-color)
Use 3 states per node: 0=unvisited, 1=visiting (in current DFS path), 2=visited. If we revisit a node in state 1, cycle detected. Classic approach.
TC: O(V+E)SC: O(V+E)
java
public boolean canFinish(int n, int[][] prereqs) {
    List<List<Integer>> adj = new ArrayList<>();
    for (int i=0;i<n;i++) adj.add(new ArrayList<>());
    for (int[] p : prereqs) adj.get(p[1]).add(p[0]);
    int[] state = new int[n]; // 0=unvisited 1=visiting 2=done
    for (int i=0;i<n;i++)
        if (!dfs(i, adj, state)) return false;
    return true;
}
private boolean dfs(int node, List<List<Integer>> adj, int[] state) {
    if (state[node] == 1) return false; // cycle!
    if (state[node] == 2) return true;
    state[node] = 1;
    for (int next : adj.get(node))
        if (!dfs(next, adj, state)) return false;
    state[node] = 2;
    return true;
}
Optimized (Kahn's BFS Topological Sort)
Compute in-degree for all nodes. Add all 0 in-degree nodes to queue. BFS: process node, reduce in-degree of neighbors; if neighbor's in-degree becomes 0, enqueue it. If processed count == n, no cycle.
TC: O(V+E)SC: O(V+E)
java
public boolean canFinish(int n, int[][] prereqs) {
    int[] inDeg = new int[n];
    List<List<Integer>> adj = new ArrayList<>();
    for (int i=0;i<n;i++) adj.add(new ArrayList<>());
    for (int[] p : prereqs) { adj.get(p[1]).add(p[0]); inDeg[p[0]]++; }
    Queue<Integer> q = new LinkedList<>();
    for (int i=0;i<n;i++) if(inDeg[i]==0) q.offer(i);
    int processed = 0;
    while (!q.isEmpty()) {
        int cur = q.poll(); processed++;
        for (int next : adj.get(cur))
            if (--inDeg[next] == 0) q.offer(next);
    }
    return processed == n;
}

Maximum Subarray (Kadane's Algorithm)
Kadane's Algorithm Banking
Problem: Given integer array nums, find the contiguous subarray with the largest sum and return its sum.
Brute Force
Try all O(n²) subarrays, compute sum for each using prefix sums or nested loop. Track the maximum sum found across all subarrays.
TC: O(n²)SC: O(1)
java
public int maxSubArray(int[] nums) {
    int max = Integer.MIN_VALUE;
    for (int i = 0; i < nums.length; i++) {
        int sum = 0;
        for (int j = i; j < nums.length; j++) {
            sum += nums[j];
            max = Math.max(max, sum);
        }
    }
    return max;
}
Optimized (Kadane's Algorithm)
Track current running sum. At each element: if current sum goes negative, reset it to 0 (start fresh from here). Update global max. Single pass, O(n).
TC: O(n)SC: O(1)
java
public int maxSubArray(int[] nums) {
    int maxSum = nums[0], currSum = nums[0];
    for (int i = 1; i < nums.length; i++) {
        currSum = Math.max(nums[i], currSum + nums[i]);
        maxSum  = Math.max(maxSum, currSum);
    }
    return maxSum;
}

Best Time to Buy and Sell Stock
Greedy / DP BankingMarket Data
Problem: Given array prices where prices[i] is the stock price on day i. Find the maximum profit from one buy and one sell (must buy before sell).
Brute Force
Try all pairs (i, j) where i < j. Compute profit = prices[j] - prices[i]. Track maximum profit.
TC: O(n²)SC: O(1)
java
public int maxProfit(int[] prices) {
    int max = 0;
    for (int i = 0; i < prices.length; i++)
        for (int j = i+1; j < prices.length; j++)
            max = Math.max(max, prices[j] - prices[i]);
    return max;
}
Optimized (One Pass Greedy)
Track the minimum price seen so far (best buy day). For each day, compute profit if sold today = price - minPrice. Update maxProfit. Single pass, no extra space.
TC: O(n)SC: O(1)
java
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE, maxProfit = 0;
    for (int price : prices) {
        minPrice  = Math.min(minPrice, price);
        maxProfit = Math.max(maxProfit, price - minPrice);
    }
    return maxProfit;
}
17
Coin Change
Dynamic Programming DP Round
Problem: Given coins of different denominations and an amount, return the fewest number of coins needed to make up that amount. Return -1 if impossible.
Brute Force (Recursion)
Recursively try every coin at each step. For each denomination, subtract it from remaining amount and recurse. Exponential due to overlapping subproblems.
TC: O(S^n)SC: O(n)
java
public int coinChange(int[] coins, int amount) {
    int res = helper(coins, amount);
    return res == Integer.MAX_VALUE ? -1 : res;
}
private int helper(int[] coins, int rem) {
    if (rem == 0) return 0;
    if (rem < 0) return Integer.MAX_VALUE;
    int min = Integer.MAX_VALUE;
    for (int coin : coins) {
        int res = helper(coins, rem - coin);
        if (res != Integer.MAX_VALUE) min = Math.min(min, res + 1);
    }
    return min;
}
Optimized (Bottom-Up DP)
Build dp array of size amount+1. dp[i] = min coins to make amount i. Initialize dp[0]=0, rest=MAX. For each amount, try all coins: dp[i] = min(dp[i], dp[i-coin]+1).
TC: O(S×n)SC: O(S)
java
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);
    dp[0] = 0;
    for (int i = 1; i <= amount; i++)
        for (int coin : coins)
            if (coin <= i)
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}
18
Word Break
DP + HashSet SprinklrProduct
Problem: Given a string s and a dictionary of strings wordDict, return true if s can be segmented into a space-separated sequence of one or more dictionary words.
Brute Force (Recursion)
Try every prefix of s. If prefix is in dictionary, recursively check if the suffix can be broken. Without memoization, results in exponential time.
TC: O(2^n)SC: O(n)
java
public boolean wordBreak(String s, List<String> wordDict) {
    return helper(s, 0, new HashSet<>(wordDict));
}
private boolean helper(String s, int start, Set<String> dict) {
    if (start == s.length()) return true;
    for (int end = start+1; end <= s.length(); end++)
        if (dict.contains(s.substring(start, end)) && helper(s, end, dict))
            return true;
    return false;
}
Optimized (Bottom-Up DP)
dp[i] = true if s[0..i) can be segmented. For each position i, check all j < i: if dp[j] is true and s[j..i) is in dictionary, dp[i] = true.
TC: O(n²)SC: O(n)
java
public boolean wordBreak(String s, List<String> wordDict) {
    Set<String> dict = new HashSet<>(wordDict);
    boolean[] dp = new boolean[s.length() + 1];
    dp[0] = true;
    for (int i = 1; i <= s.length(); i++)
        for (int j = 0; j < i; j++)
            if (dp[j] && dict.contains(s.substring(j, i))) {
                dp[i] = true; break;
            }
    return dp[s.length()];
}

Rotting Oranges
Multi-source BFS BFS Maturity Test
Problem: Grid of 0 (empty), 1 (fresh orange), 2 (rotten). Every minute, rotten oranges infect adjacent (4-directional) fresh ones. Return the minimum minutes until no fresh oranges remain. Return -1 if impossible.
Brute Force (Simulation)
Simulate minute by minute. In each pass, find all rotten oranges and rot their fresh neighbors. Count passes until no new oranges are rotted. Stop or detect impossibility.
TC: O((m×n)²)SC: O(m×n)
java
public int orangesRotting(int[][] grid) {
    int m = grid.length, n = grid[0].length, minutes = 0, fresh = 0;
    for (int[] r : grid) for (int v : r) if(v==1) fresh++;
    while (true) {
        boolean changed = false;
        int[][] copy = new int[m][n];
        for (int[] row : grid) copy[minutes % m] = row.clone(); // deep copy
        // simplified: mark newly rotten in a temp matrix
        int[][] temp = new int[m][n];
        for(int i=0;i<m;i++) temp[i]=grid[i].clone();
        int[] dr={0,0,1,-1},dc={1,-1,0,0};
        for(int i=0;i<m;i++) for(int j=0;j<n;j++) if(grid[i][j]==2)
            for(int d=0;d<4;d++){int ni=i+dr[d],nj=j+dc[d];
                if(ni>=0&&ni<m&&nj>=0&&nj<n&&grid[ni][nj]==1){temp[ni][nj]=2;fresh--;changed=true;}}
        if(!changed) break;
        grid=temp; minutes++;
    }
    return fresh==0 ? minutes : -1;
}
Optimized (Multi-source BFS)
Add ALL initially rotten oranges to queue simultaneously (multi-source). BFS level-by-level — each level = 1 minute. Count fresh oranges; decrement as they rot. Time = BFS depth.
TC: O(m×n)SC: O(m×n)
java
public int orangesRotting(int[][] grid) {
    int m = grid.length, n = grid[0].length, fresh = 0, minutes = 0;
    Queue<int[]> q = new LinkedList<>();
    for (int i=0;i<m;i++) for(int j=0;j<n;j++) {
        if (grid[i][j]==2) q.offer(new int[]{i,j});
        else if (grid[i][j]==1) fresh++;
    }
    int[] dr={0,0,1,-1}, dc={1,-1,0,0};
    while (!q.isEmpty() && fresh > 0) {
        minutes++;
        for (int sz = q.size(); sz > 0; sz--) {
            int[] cur = q.poll();
            for (int d=0;d<4;d++) {
                int nr=cur[0]+dr[d], nc=cur[1]+dc[d];
                if(nr>=0&&nr<m&&nc>=0&&nc<n&&grid[nr][nc]==1)
                    { grid[nr][nc]=2; fresh--; q.offer(new int[]{nr,nc}); }
            }
        }
    }
    return fresh == 0 ? minutes : -1;
}

Trapping Rain Water
Two Pointers Optimization Thinking
Problem: Given height array representing elevation map (bar width = 1), compute how much water it can trap after raining.
Brute Force
For each bar i, scan left to find maxLeft, scan right to find maxRight. Water at i = min(maxLeft, maxRight) - height[i]. Sum all positive values.
TC: O(n²)SC: O(1)
java
public int trap(int[] height) {
    int n = height.length, water = 0;
    for (int i = 1; i < n-1; i++) {
        int maxL = 0, maxR = 0;
        for (int j=0; j<=i; j++) maxL = Math.max(maxL, height[j]);
        for (int j=i; j<n; j++) maxR = Math.max(maxR, height[j]);
        water += Math.min(maxL, maxR) - height[i];
    }
    return water;
}
Better (Precompute prefix/suffix max arrays)
Precompute leftMax[i] and rightMax[i] arrays in O(n). Then for each position, water = min(leftMax[i], rightMax[i]) - height[i]. Two extra arrays eliminate re-scanning.
TC: O(n)SC: O(n)
java
public int trap(int[] height) {
    int n = height.length;
    int[] leftMax = new int[n], rightMax = new int[n];
    leftMax[0] = height[0];
    for (int i=1;i<n;i++) leftMax[i] = Math.max(leftMax[i-1], height[i]);
    rightMax[n-1] = height[n-1];
    for (int i=n-2;i>=0;i--) rightMax[i] = Math.max(rightMax[i+1], height[i]);
    int water = 0;
    for (int i=0;i<n;i++) water += Math.min(leftMax[i], rightMax[i]) - height[i];
    return water;
}
Optimized (Two Pointers)
Use left and right pointers. The side with smaller max height determines water level (bounded by lower side). Move inward: if left height ≤ right height, compute water for left and move left++; else move right--. O(1) space.
TC: O(n)SC: O(1)
java
public int trap(int[] height) {
    int left = 0, right = height.length - 1;
    int maxL = 0, maxR = 0, water = 0;
    while (left < right) {
        if (height[left] <= height[right]) {
            if (height[left] >= maxL) maxL = height[left];
            else water += maxL - height[left];
            left++;
        } else {
            if (height[right] >= maxR) maxR = height[right];
            else water += maxR - height[right];
            right--;
        }
    }
    return water;
}
